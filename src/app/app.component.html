Learning and Implementing Angular

Installing and Building Angular

The process overall of installing Angular was fairly intuitive; making use of tutorials from Geeks for Geeks and Angular.IO provided a comprehensive process on how to get the framework up and running, despite a lack of experience using components and Angular as a whole. That being said, the overall installation was not the smoothest, which will be covered in greater detail in the following section.

The steps that were taken to create an Angular app were very simple, comprising of the following commands:

   ng new my-app   -   (Creates the app)
   ng serve   -   (Runs the app)

Difficulties

Whilst the installation process was straight forward due to the resources I used, there was still difficulties I faced trying to get it isntalled. When trying to install Angular through the command line, I was faced with an error, due to not having an NPM packet manager (like nvm). The path only got deeper from there, as trying to install nvm through the command line returned another error, as I did not have bash installed either. Thus, this installation process resulted in a repetitive loop of installing the prerequisite software needed to install more prerequisite software to install Angular, all whilst having the restart my computer each time to register the change.

Additionally, I wanted to implement a small component to demonstrate an understanding of Angular. Originally aiming to use a clicker component, I ultimately was unable to get it working, having the component logic and stylizing setup but unable to have it present on this HTML page. Ultimately, I opted to exclude the faulty counter due to time constrictions and upcoming exams, and aim to play around with Angular in the future to get it working in the future.

After completing this webpage and attempting to host it, I ran into yet another issue; having a 404: Page not Found error trying to host the app on GitHub pages. Ultimately (and evidently), the issue was resolved due to trial and error and online forums, where I had to move the index.html file into the docs folder.
